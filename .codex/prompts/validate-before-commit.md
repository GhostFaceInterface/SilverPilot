# Validate Before Commit

Use `.codex/workflows/aggressive-validation.md` and `.codex/workflows/commit-readiness.md`.

Required response:
1. Changed files and intended scope.
2. Applicable validation levels.
3. Commands to run or already run.
4. Evidence collected.
5. Missing evidence.
6. Commit readiness: PASS / FAIL / UNKNOWN.

Do not commit without explicit user approval.
